# Concept

![Img](https://studio-assets.supernova.io/design-systems/161625/39c0dc6b-ca18-442d-8bdb-95d288d8982b.png)

Local libraries are Figma files with published components relevant only to a Tribe (e.g., Convert), a Squad, a Brand (e.g., Fairmont), or specially tailored cases. 

Please check the [complete library structure](../first-steps/library-guidelines.md) if you want to learn more about individual, candidate or welcome libraries for both web and app platforms.

> undefined  
> If you want to start a local library or adjust the scope of yours, reach out to your Design System Manager (DSM)!

# **What makes a component local?**

Local components are shared across multiple contexts within a scope but typically dont meet the Core inclusion criteria [inclusion criteria](../../contributing/contribution-model/presentation.md) of agnosticism. Heres how they generally compare:

- **Reusable**: By nature, local components are reusable, shared by at least two designers or two separate projects.

- **Stable**: Stability isnt a requirement; local components can update rapidly since they are closely tied to the business.

- **Agnostic**: Local components are usually specific to their context, using unique data or steps in a user flow. For example, a Card displaying hotel results for a search is specific to the Convert library.

- **Simple**: Reusable components work best when simple.

- **Common**: Local components are often uncommon, which is intentional.


Any designer can [start, contribute to, and update](./maintain-your-library.md) local libraries, as long as it fits within their scope.
